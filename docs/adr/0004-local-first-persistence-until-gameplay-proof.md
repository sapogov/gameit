# Local-First Persistence Until Gameplay Proof

Status: accepted

GameIt Monsters will keep early gameplay persistence local-first and behind repository interfaces until the core loops are proven: onboarding, Cards/Eggs, Wild Encounters, Battles, Farms, Theft, and Station travel. Production auth, database, and hosting choices are deferred so gameplay iteration is not blocked by infrastructure, while repository boundaries keep a future move to account-backed persistence explicit.
